# 更新日志

[English](CHANGELOG.md)

本文件记录项目的重要版本变更。

## [Unreleased]

### Added

- 集中式 TCP 内网穿透：一个 `tunnel-server`、多个 `tunnel-client`，并支持通过 Web 后台或 CLI 管理路由
- 内网穿透管理页面及相关 API
- 面向开源项目的双语文档入口

### Changed

- 将公开文档收敛为更精简、更适合开源项目的结构
- 将内网穿透工作流从“每端口单独命令”调整为“客户端常驻连接 + 服务端管理路由”
- Web 管理界面文档统一为默认随机端口启动说明

### Fixed

- 强化了 tunnel 的 TLS 优先策略、握手限制、消息大小限制和待配对连接上限
- 修正了主文档中过时的命令示例和日志说明

## [1.4.0] - 2026-01-18

### Added

- SOCKS5 认证缓存
- HTTP 连接池与更稳定的 DNS 缓存行为
- 超时类配置热更新
- 集中常量配置与数据库连接池设置

### Changed

- 降低了配置热重载压力，清理了重复的启动逻辑
- 提升了代理启动和运行时配置代码的可维护性

### Fixed

- 监听器关闭处理与错误退避逻辑
- 较高并发下 SOCKS5 / HTTP 运行稳定性问题

## [1.3.0] - 2026-01-17

### Added

- 代理目标的 SSRF 防护
- 凭证校验中的时序攻击缓解
- SOCKS5 请求中的 IPv6 支持

### Changed

- 用户模型统一为用户名全局唯一
- 简化了删除用户语义及相关 API

### Fixed

- SOCKS5 协议校验缺口
- 长连接场景下的 HTTP 代理请求处理问题
- 用户与白名单持久化的并发竞争问题

## [1.2.1] - 2026-01-17

### Changed

- 调整连接超时策略，避免长请求被过早终止
- 在持久化用户数据中强制用户名全局唯一

### Fixed

- 连接建立超时处理
- HTTP 请求体转发问题
- 写错误未检查与可能触发 panic 的类型断言

## [1.2.0] - 2026-01-17

### Changed

- 将 Windows 快捷方式创建从脚本方式改为纯 Go 实现
- 降低了启动快捷方式相关的 Windows 安全软件误报概率

## [1.1.0] - Previous Release

### Changed

- 自动启动从注册表切换为启动文件夹快捷方式
- 增加 Windows 资源元数据并改进构建自动化

## [1.0.0] - Initial Release

### Added

- SOCKS5 与 HTTP/HTTPS 代理支持
- 用户名/密码认证和 IP 白名单
- Web 管理界面与 Windows 托盘模式
- 基于 SQLite 的持久化
