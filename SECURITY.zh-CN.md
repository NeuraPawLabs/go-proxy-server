# 安全策略

[English](SECURITY.md)

## 重点关注范围

以下模块的安全问题尤其值得优先报告：

- 认证与凭证处理
- IP 白名单逻辑
- 代理请求校验与 SSRF 防护
- 内网穿透鉴权、TLS 与路由隔离
- 仅监听本地的 Web 管理接口

## 如何报告漏洞

如果你认为发现了安全漏洞，请先私下联系维护者，不要第一时间公开提 issue。

建议提供：

- 受影响的版本或提交
- 风险影响说明
- 复现步骤或 PoC
- 可行的缓解建议（如果有）

## 披露建议

- 请给维护者留出复现、修复和验证的时间。
- 在修复或缓解方案可用前，避免公开披露细节。
- 涉及安全行为变化时，应同步更新相关文档。
