# 贡献指南

[English](CONTRIBUTING.md)

感谢你关注 Go Proxy Server。

## 开发流程

1. Fork 仓库并创建功能分支。
2. 保持改动聚焦，提交信息清晰。
3. 本地完成基本验证：
   - `go test ./...`
   - 如果改动了前端，执行 `cd web-ui && npm run build`
4. 如果行为、命令或 API 有变化，请同步更新文档。
5. 提交 Pull Request 时说明：
   - 改了什么
   - 为什么这么改
   - 如何验证

## 贡献约定

- 一个 PR 尽量只解决一类问题，避免混入无关重构。
- 除非需求明确要求，不要随意改变现有平台行为。
- 优先保证代码可维护性与可读性。
- 新增命令或 API 时，请同时更新中英文文档入口。

## 提交 Bug 时建议提供

- 操作系统和版本
- 启动命令
- 相关配置
- 日志或截图
- 复现步骤

## 安全问题

如果是安全漏洞，请不要先公开提交 issue，参考 [SECURITY.zh-CN.md](SECURITY.zh-CN.md)。
